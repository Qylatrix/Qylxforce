# Contributing to Qylatrix Offensive Suite

Thank you for your interest in contributing! 🎉

## Getting Started

1. **Fork** the repository and create your branch from `main`:
   ```bash
   git checkout -b feature/my-new-feature
   ```

2. **Install** development dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. **Make your changes** and ensure the code is clean and documented.

4. **Test** your changes thoroughly — run the CLI and web app to verify.

5. **Commit** with a clear message:
   ```bash
   git commit -m "feat: add new mutation algorithm to dictionary forge"
   ```

6. **Push** and open a **Pull Request** against `main`.

## Code Style

- Follow **PEP 8** for Python code
- Use descriptive variable and function names
- Add docstrings to all public functions and classes
- Keep functions focused and well-scoped

## Reporting Issues

- Use the GitHub Issues tracker
- Include steps to reproduce, expected vs. actual behavior, and your Python version

## Ethical Guidelines

This is an educational security tool. All contributions must:
- Be strictly within ethical and legal boundaries
- Not introduce functionality that enables unauthorized access to real systems
- Include appropriate disclaimers where necessary
